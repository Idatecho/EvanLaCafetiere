        # Crée une fenêtre avec OpenGL activé et double buffering
        pg.display.set_mode(WIN_RES, flags=pg.OPENGL | pg.DOUBLEBUF)

(i) :

🎞️ Qu’est-ce que le buffering (mise en mémoire tampon en FR) ?

Le buffering, c’est l’usage d’une zone mémoire (buffer) pour stocker temporairement ce qui doit être affiché à l’écran. En rendu graphique, cela signifie qu’on dessine d’abord dans un buffer hors écran, puis on affiche ce contenu en une seule fois.

Pourquoi ? Parce qu’afficher l’image pixel par pixel en direct créerait des effets de "scintillement", de "déchirure" ou d’images incomplètes.

📦 Qu’est-ce que le Z-buffer ?

Le Z-buffer (aussi appelé depth buffer) est une mémoire spéciale utilisée pour le rendu 3D, qui enregistre la profondeur de chaque pixel à l’écran (sa distance à la caméra).

  • Lors du rendu, pour chaque pixel dessiné, OpenGL compare sa profondeur avec celle déjà enregistrée à cette position dans le Z-buffer.
  • Si le nouveau pixel est plus proche de la caméra, il est affiché et sa profondeur remplace l’ancienne.
  • Sinon, il est ignoré (car caché par un objet devant lui).

🧠 Pourquoi c’est essentiel en 3D ?

Sans Z-buffer, les objets s’afficheraient dans l’ordre de dessin, donc un objet loin pourrait recouvrir un objet proche = rendu incorrect.

Avec le Z-buffer :

  • Le rendu respecte la profondeur réelle.
  • Tu peux dessiner les objets dans n’importe quel ordre, le Z-buffer fait le tri.

❓ Et pourquoi 24 bits ?

Un Z-buffer peut avoir :

  • 16 bits (moins précis)
  • 24 bits (standard et assez précis pour la plupart des scènes 3D)
  • 32 bits (encore plus précis, mais plus coûteux en ressources)

Ici, 24 signifie que chaque pixel a 2²⁴ (≈ 16 millions) de niveaux de profondeur possibles entre "proche" et "loin". C’est un bon compromis entre précision et performance.

🧠 Et le double buffering, alors ?

Le double buffering consiste à utiliser deux buffers :

  • 🔲 Back buffer (tampon arrière) : on y dessine la prochaine image.
  • 🟦 Front buffer (tampon avant) : c’est ce qui est actuellement affiché à l’écran.

Une fois l’image complètement dessinée dans le back buffer, on fait un flip ou swap :

👉 les deux buffers échangent leur rôle, et l’image complète s’affiche d’un coup.

✅ Avantages du double buffering :

  • ✅ Pas de scintillement
  • ✅ Pas de tearing (déchirure de l’image)
  • ✅ Rendu fluide et propre, surtout avec des FPS élevés

       # Crée un contexte OpenGL moderne via moderngl
        self.ctx = mgl.create_context()

        # Active certains comportements graphiques :
        # - DEPTH_TEST : gestion de la profondeur (évite que des objets derrière s'affichent devant)
        # - CULL_FACE : supprime les faces non visibles (face culling)
        # - BLEND : permet la transparence
        self.ctx.enable(flags=mgl.DEPTH_TEST | mgl.CULL_FACE | mgl.BLEND)

        # Mode de gestion de la mémoire de moderngl (automatique ici)
        self.ctx.gc_mode = 'auto'

        # Horloge pour gérer le temps entre les frames
        self.clock = pg.time.Clock()
        
        # Temps écoulé depuis la dernière frame
        self.delta_time = 0
        
        # Temps total écoulé depuis le lancement du jeu (en secondes)
        self.time = 0

        # Variable pour garder la boucle principale active
        self.is_running = True

On en a finit pour l’initialisation dans la class. Créons maintenant une fonction pour mettre à jour le temps :

 # Met à jour le temps
    def update(self):
        # Récupère le temps écoulé depuis la dernière frame (en ms)
        self.delta_time = self.clock.tick()
        
        # Récupère le temps total depuis le début du programme (en secondes)
        self.time = pg.time.get_ticks() * 0.001
        
        # Affiche les FPS dans le titre de la fenêtre
        pg.display.set_caption(f'{self.clock.get_fps() :.0f}')

Créons maintenant une fonction pour effectuer le rendu :

    # Effectue le rendu (nettoie l'écran et échange les buffers)
    def render(self):
        # Vide le tampon de couleur et de profondeur (efface l’écran)
        self.ctx.clear()
        
        # Affiche le contenu rendu à l'écran (double buffering)
        pg.display.flip()

Une fonction pour gérer le clavier et la fenêtre :

 # Gère les événements clavier et fenêtre
    def handle_events(self):
        for event in pg.event.get():
            # Si l'utilisateur ferme la fenêtre ou appuie sur Échap, arrêter le programme
            if event.type == pg.QUIT or (event.type == pg.KEYDOWN and event.key == pg.K_ESCAPE):
                self.is_running = False

Une fonction pour la boucle principale :

    # Boucle principale du moteur
    def run(self):
        while self.is_running:
            self.handle_events()  # Gérer les entrées utilisateur
            self.update()         # Mettre à jour le temps et l’état
            self.render()         # Dessiner l’image à l’écran
        
        # Si on sort de la boucle, quitter proprement
        pg.quit()
        sys.exit()
